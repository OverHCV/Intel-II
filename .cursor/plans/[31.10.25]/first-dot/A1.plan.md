# Student Performance Analysis - Architecture Plan

> **Objective**: Complete third partial exam with modular, reusable architecture
> **Philosophy**: Clean separation (Logic | Data | UI), tech-agnostic design, pattern-first approach

---

## CONTEXT

**Problem Domain**: Educational data analysis (classification + clustering)

**Dataset Strategy**: 
- **Math Dataset (student-mat.csv)**: 395 students → TRAINING SET (more samples)
- **Portuguese Dataset (student-por.csv)**: 649 students → TEST SET (cross-domain validation)
- **Common Features**: 33 attributes (mixed types), same structure
- **Target**: G3 (0-20) → categorical transformation required
- **Risk**: G1, G2 correlation (data leakage) → exclude from features

**Cross-Domain Prediction Philosophy**:
> Train on Math, test on Portuguese as a "leap of faith" to evaluate if patterns learned 
> from one academic subject generalize to another. Reflect on what works and what doesn't.

**Deliverables**:
1. Decision Trees + rule analysis (0.9 pts)
2. Hierarchical clustering + J4 analysis (0.9 pts)  
3. K-means + comparison (0.7 pts)
4. Jupyter notebook + oral presentation (2.5 pts)

**Core Principle**: Build architecture where **next tab reuses layout pattern but swaps content/logic**

---

## ARCHITECTURE PRINCIPLES

```apl
VERB define_architecture_principles OUT(principles: obj) {
    
    separation_of_concerns: {
        rule: "Logic NEVER knows about UI, UI NEVER contains logic",
        layers: {
            LOGIC: "Pure algorithms, no display code",
            DATA: "Loading, transformation, validation",
            UI: "Display, interaction, layout patterns"
        },
        communication: "Through well-defined contracts (interfaces)"
    }
    
    modularity: {
        rule: "Each module < 300 LOC, single responsibility",
        file_organization: {
            by_layer: "logic/, data/, ui/ - NOT by feature",
            by_responsibility: "One file = one clear purpose"
        }
    }
    
    reusability: {
        WHAT_CHANGES: {
            logic: "Algorithm implementations vary per task",
            content: "Visualizations, metrics, parameters differ",
            data_transforms: "Preprocessing may differ"
        },
        WHAT_STAYS_SAME: {
            layout_patterns: "Two-column, tabs, controls structure",
            ui_components: "Buttons, sliders, displays",
            data_pipeline: "Load → validate → transform flow",
            state_management: "How state is stored/accessed"
        },
        pattern: "Define TEMPLATE with SLOTS for variable content"
    }
    
    testability: {
        rule: "Each layer independently testable",
        logic_tests: "Test algorithms with mock data",
        data_tests: "Test transformations with sample inputs",
        ui_tests: "Test rendering with mock logic"
    }
    
    extensibility: {
        rule: "Adding new task = plug new logic into existing structure",
        example: "Task 4 would reuse ALL infrastructure, only swap logic module"
    }
}
```

## LAYER DEFINITIONS

### Layer 1: DATA (Pure Transformation)

```apl
VERB define_data_layer OUT(@data/) {
    
    responsibility: "Load, validate, transform data - NO business logic",
    
    INTERFACE DataProvider {
        VERB load IN(source: str, options: obj) OUT(raw_data: dataframe) 
        VERB validate IN(raw_data: dataframe) OUT(is_valid: bool, errors: list)
        VERB transform IN(raw_data: dataframe, strategy: obj) OUT(processed_data: obj)
    }
    
    modules: {
        loader_py: {
            purpose: "File I/O only",
            functions: [
                "load_csv(filepath) → dataframe",
                "load_dataset(name: 'math' | 'portuguese') → dataframe",
                "save_snapshot(data, version_id) → None"
            ],
            no_logic: "Just read/write, no decisions"
        },
        
        validator_py: {
            purpose: "Data quality checks",
            functions: [
                "check_schema(df, expected_cols) → bool",
                "check_missing(df, tolerance) → bool",
                "check_ranges(df, constraints) → bool",
                "validate_cross_dataset(df_train, df_test) → alignment_report"
            ]
        },
        
        transformer_py: {
            purpose: "Data transformations",
            functions: [
                "encode_categorical(df, method) → encoded_df",
                "scale_numerical(df, method) → scaled_df",
                "engineer_target(df, strategy) → target_array",
                "split_features_target(df, target_col) → X, y",
                "balance_classes(X, y, method) → X_balanced, y_balanced"
            ],
            parametrized: "Strategy pattern - caller decides HOW"
        },
        
        balancer_py: {
            purpose: "Class balancing strategies",
            functions: [
                "apply_smote(X, y, params) → X_resampled, y_resampled",
                "apply_random_oversample(X, y) → X_resampled, y_resampled",
                "apply_random_undersample(X, y) → X_resampled, y_resampled",
                "get_class_distribution(y) → distribution_dict"
            ],
            note: "Used by Dataset Review tab"
        }
    },
    
    contract: {
        input: "File path + transformation options",
        output: "X (features), y (target), metadata",
        guarantees: [
            "X.shape[0] == y.shape[0]",
            "No NaN in output",
            "Metadata contains feature names, types, stats"
        ]
    }
}
```

### Layer 2: LOGIC (Pure Algorithms)

```apl
VERB define_logic_layer OUT(@logic/) {
    
    responsibility: "Implement ML algorithms - NO data loading, NO display",
    
    INTERFACE Algorithm {
        VERB fit IN(X: array, y: array, params: obj) OUT(model: obj)
        VERB predict IN(model: obj, X: array) OUT(predictions: array)
        VERB evaluate IN(y_true: array, y_pred: array) OUT(metrics: obj)
        VERB explain IN(model: obj) OUT(explanation: obj)
    }
    
    modules: {
        decision_tree_py: {
            purpose: "Tree algorithms",
            functions: [
                "train_cart(X, y, hyperparams) → model",
                "extract_rules(model) → list[rule_obj]",
                "rank_rules(rules, criteria) → sorted_rules",
                "cross_validate(X, y, model_fn, k) → cv_scores"
            ],
            pure: "Takes arrays, returns objects - no side effects"
        },
        
        clustering_py: {
            purpose: "Clustering algorithms",
            functions: [
                "hierarchical(X, linkage) → linkage_matrix",
                "kmeans(X, k, params) → model, labels",
                "calculate_j4(X, labels) → float",
                "compare_clusterings(labels1, labels2) → agreement_metrics"
            ]
        },
        
        analysis_py: {
            purpose: "Result analysis",
            functions: [
                "find_optimal_k(linkage_matrix, X, criterion) → k, scores",
                "analyze_rules(rules) → insights",
                "cluster_statistics(X, labels) → stats_per_cluster"
            ]
        }
    },
    
    contract: {
        input: "Numpy arrays + parameters",
        output: "Models, predictions, metrics (no formatting)",
        guarantees: [
            "Deterministic (same input → same output)",
            "No file I/O",
            "No print statements",
            "Returns data, not strings"
        ]
    }
}
```

### Layer 2.5: VERSIONING (Experiment Timeline Management)

```apl
VERB define_versioning_layer OUT(@versioning/) {
    
    responsibility: "Manage experiment history, versions, and timeline - NO algorithms, NO UI",
    
    philosophy: {
        not_just_cache: "Not a simple .cache folder - a proper module for version control",
        timeline_oriented: "Track experiments as a timeline, not scattered files",
        comparable: "Enable easy comparison across different experiment versions"
    },
    
    INTERFACE VersionManager {
        VERB save_experiment IN(experiment: obj, metadata: obj) OUT(version_id: str)
        VERB load_experiment IN(version_id: str) OUT(experiment: obj)
        VERB list_experiments IN(filters: obj) OUT(list[experiment_summary])
        VERB compare_experiments IN(version_ids: list[str]) OUT(comparison_report: obj)
    }
    
    modules: {
        experiment_store_py: {
            purpose: "Persist and retrieve experiments",
            functions: [
                "save(experiment_data, metadata) → version_id",
                "load(version_id) → experiment_data",
                "delete(version_id) → bool",
                "exists(version_id) → bool"
            ],
            storage_strategy: {
                format: "JSON for metadata + pickle for models/arrays",
                structure: "experiments/{algorithm_type}/{version_id}/",
                files: [
                    "metadata.json - timestamp, params, metrics",
                    "model.pkl - trained model object",
                    "predictions.npy - prediction arrays",
                    "visualizations/ - saved plots as PNG"
                ]
            }
        },
        
        timeline_manager_py: {
            purpose: "Manage experiment timeline and history",
            functions: [
                "get_timeline(algorithm_type, limit) → chronological_list",
                "filter_by_date(start, end) → filtered_experiments",
                "filter_by_metric(metric_name, threshold) → filtered_experiments",
                "get_best(criterion) → best_experiment",
                "get_recent(n) → last_n_experiments"
            ],
            features: [
                "Chronological ordering",
                "Filter by date range",
                "Filter by metric thresholds",
                "Search by parameters"
            ]
        },
        
        comparison_engine_py: {
            purpose: "Compare multiple experiments",
            functions: [
                "compare_metrics(version_ids) → comparison_table",
                "compare_parameters(version_ids) → diff_report",
                "generate_comparison_plots(version_ids) → matplotlib_figures",
                "calculate_improvement(baseline_id, new_id) → delta_metrics"
            ],
            outputs: [
                "Side-by-side metric tables",
                "Parameter diff reports",
                "Timeline charts (metric evolution)",
                "Performance comparison plots"
            ]
        },
        
        version_id_generator_py: {
            purpose: "Generate unique, sortable version IDs",
            strategy: "timestamp_algorithmtype_hash",
            example: "20251101_143022_svm_a3f2d1",
            benefits: [
                "Chronologically sortable",
                "Self-documenting (includes algorithm)",
                "Unique (hash prevents collisions)"
            ]
        }
    },
    
    data_structures: {
        ExperimentRecord: {
            version_id: "string (unique identifier)",
            timestamp: "ISO 8601 datetime",
            algorithm_type: "svm | ann | hierarchical | kmeans",
            dataset_info: {
                source: "math | portuguese | both",
                n_samples: "int",
                n_features: "int",
                class_distribution: "dict"
            },
            parameters: {
                hyperparam_1: "value",
                hyperparam_2: "value"
            },
            metrics: {
                metric_1: "value",
                metric_2: "value"
            },
            artifacts: {
                model_path: "str",
                predictions_path: "str",
                visualizations_paths: "list[str]"
            },
            notes: "string (optional user notes)"
        }
    },
    
    contract: {
        input: "Experiment results from logic layer",
        output: "Version IDs, experiment lists, comparison reports",
        guarantees: [
            "Version IDs are unique and sortable",
            "Timeline is always chronological",
            "All experiments are recoverable",
            "Comparisons use consistent metrics"
        ]
    },
    
    ui_integration: {
        experiment_table: "Display all experiments in sortable table",
        timeline_chart: "Matplotlib line chart showing metric evolution over time",
        comparison_view: "Side-by-side comparison of selected experiments",
        best_model_highlight: "Visually distinguish best performing experiment"
    }
}
```

```apl
VERB define_ui_layer OUT(@ui/) {

                "param_selectors - different per algorithm",
                "viz1, viz2 - different visualizations",
                "metrics1, metrics2 - different metrics"
            ],
            constant: "Layout structure, interaction flow"
        }
    },
    
    modules: {
        components: {
            layouts_py: "TwoColumn, Tabs, Grid",
            controls_py: "Sliders, Selects, Buttons",
            displays_py: "MetricCard, Table, PlotContainer"
        },
        
        pages: {
            template_py: {
                purpose: "Abstract page template",
                methods: [
                    "render_controls() - override per page",
                    "render_results() - override per page",
                    "handle_train_click() - calls logic layer"
                ]
            },
            
            dataset_review_py: "Extends template, dataset manipulation & balancing",
            decision_tree_page_py: "Extends template, provides content",
            hierarchical_page_py: "Extends template, provides content",
            kmeans_page_py: "Extends template, provides content",

            purpose: "Wire layers together",

                user_clicks_train()
                  → gather_params_from_ui()
                  → data = DataProvider.load_and_transform(params)
                  → results = Algorithm.fit_and_evaluate(data, params)
                  → version_id = VersionManager.save_experiment(results)

        },
        
        state_manager_py: {
            purpose: "Centralized state management",
            responsibility: "Manage application state persistence across UI interactions",

                local_state: {
                    scope: "Widget-level, lost on rerun",
                    use_for: [
                        "Form inputs",
                        "Temporary selections",
                        "UI-only values"
                    ]
                },
                
                persistent_state: {
                    scope: "Survives app restarts",
                    use_for: [
                        "Experiment history",
                        "Best models",
                        "User preferences"
                    ],
                    storage: "Managed by versioning layer"
                }
            },
            
            functions: [
                "init_state(defaults) → None",
                "get_state(key, default) → value",
                "set_state(key, value) → None",
                "reset_state(keys) → None",
                "state_exists(key) → bool"

            patterns: {

                    why: "Don't compute until needed",
                    example: `

                },
                
                versioned_state: {
                    why: "Track state changes over time",
                    example: `
                        state["dataset_v1"] = original_data
                        state["dataset_v2"] = balanced_data
                        state["active_version"] = "v2"
                    `
                },
                
                cross_page_state: {
                    why: "Share data between pages/tabs",
                    example: `
                        # Dataset Review tab sets:
                        state["prepared_data"] = (X, y)
                        
                        # Decision Tree tab reads:
                        X, y = state.get("prepared_data", load_default())
                    `
                }
            }
        }
    },
    
    contract: {
        input: "Data objects from logic layer (models, metrics, arrays)",
        output: "Visual representation",
        responsibilities: [
            "Format data for display",
            "Handle user interactions",
            "Manage UI state",
            "Call logic layer functions"
        ],
        prohibited: [
            "Implement algorithms",
            "Load raw files",
            "Transform data"
        ]
    }
}
```
