### BUG #1: Import Resolution Failure ❌ CRITICAL

**Error**: `ModuleNotFoundError: No module named 'config'`

**Location**: All files in `ui/` trying to import from root-level modules

**Root Cause**:
```
ui/app.py tries: from config import CONF, Keys
Python looks in: ui/ directory (where app.py is)
Config is at: ./config.py (project root)
Result: ModuleNotFoundError
```

**Why it happened**:
1. **No ENVIRONMENT_CHECK**: Didn't verify how Streamlit resolves imports when running `ui/app.py`
2. **No IMPORT_TRACE**: Didn't trace Python's module search path for nested file structure
3. **Assumed happy path**: Assumed imports would work because files exist in project
4. **Framework-blind**: Treated Streamlit like normal Python, but it has different import context

**Impact**: App crashes immediately on startup - complete failure

**Time wasted**: Would have taken 30 seconds to prevent with proper checks

## LESSONS_LEARNED

### Lesson #1: Framework Execution Context ≠ Normal Python

**Pattern**: Different frameworks change how Python resolves imports

**Examples**:
- Streamlit running `ui/app.py` may set cwd differently
- Django apps expect specific project structure
- Flask blueprints have import resolution rules
- Jupyter notebooks have different sys.path

**Rule**: Always TRACE import resolution for framework-specific execution

### Lesson #2: Validate Before "Working Prototype"

**What I said**: "Let's test the app!"
**What I did**: Ran command without waiting for result
**What I should do**: Wait for terminal output, verify it works, THEN celebrate

**Rule**: VALIDATE means "confirm it actually works", not "assume it will work"

### Lesson #3: Pessimistic Path Planning

**Current approach**: Assume happy path, fix when breaks
**Better approach**: Assume things will break, prevent before they happen

**Checklist for ANY new file structure**:
- [ ] IMPORT_TRACE: Verify module resolution
- [ ] ENVIRONMENT_CHECK: Confirm runtime context
- [ ] TEST_EXECUTION: Run before declaring done
- [ ] FAILURE_MODES: What can go wrong?

### Lesson #4: "Three Layers Rule" Applies to Execution Too

**Data flow**: Component → Storage → Database ✅
**Execution flow**: Code → Framework → Runtime ❌ (I didn't trace this!)

**Should apply same rigor**:
1. **Code Layer**: What imports are written
2. **Framework Layer**: How framework resolves them (Streamlit context)
3. **Runtime Layer**: Where Python actually finds modules (sys.path)

## NEW PARLANG VERBS NEEDED

### ENVIRONMENT_CHECK (Level 2 - Analysis)

**Purpose**: Verify framework/runtime environment before execution

```
ENVIRONMENT_CHECK IN(@file_structure, @framework) OUT(env_validation): {
  framework: "Streamlit | Django | Flask | etc",
  execution_command: "streamlit run ui/app.py",
  checks: {
    working_directory: "Where is cwd when command runs?",
    module_search_path: "What's in sys.path?",
    import_resolution: "Can nested files import root modules?",
    config_location: "Where does framework look for configs?"
  },
  failure_modes: [
    "ModuleNotFoundError if paths wrong",
    "Import resolution differs from normal Python",
    "Framework sets different cwd than expected"
  ],
  why: "Framework execution context often differs from normal Python"
}
```

**When to use**: ALWAYS before running framework-specific code

### IMPORT_TRACE (Level 2 - Analysis)

**Purpose**: Trace how Python will resolve imports in given structure

```
IMPORT_TRACE IN(@project_structure, @execution_point) OUT(import_map): {
  executing_from: "ui/app.py",
  trying_to_import: "config",
  python_searches: [
    "1. Same directory (ui/)",
    "2. Parent directories in sys.path",
    "3. Site-packages"
  ],
  will_find: "YES at ./config.py" OR "NO - ModuleNotFoundError",
  solution_if_no: [
    "Add sys.path.append(os.path.dirname(__file__)/../)",
    "Use relative imports",
    "Run from different directory",
    "Set PYTHONPATH"
  ],
  why: "Imports fail if Python can't find module in search path"
}
```

**When to use**: ALWAYS when creating nested file structures with cross-directory imports

### FRAMEWORK_CHECKS (Enhanced)

Add to existing FRAMEWORK_CHECKS:

STREAMLIT_CHECKS IN(@ui/, @app.py) OUT(validation): {

### BUG #2: Namespace Collision & Import Failure ❌ CRITICAL

**Error**: `ModuleNotFoundError` in `ui/pages/__init__.py` trying to import `config_page` from `ui.pages.config`

**Location**: File renamed from `config.py` to `conf.py` but imports not updated

**Root Cause - ORGANIZATIONAL FAILURE**:
```
CREATED: ui/pages/config.py (has config_page function)
CONFLICT: Root-level config.py already exists
RENAME: User renamed to conf.py to avoid conflict
FORGOT: Update ui/pages/__init__.py imports
RESULT: Import breaks, app crashes
```

**Why it happened - PLANNING GAPS**:
1. **No NAMESPACE_CHECK**: Didn't verify name uniqueness before creating files
2. **No DEPENDENCY_TRACE**: Didn't track what imports this file after creating it
3. **No POST-CHANGE_VALIDATION**: Didn't verify system still works after rename
4. **Reactive not Proactive**: Waited for user to rename instead of detecting conflict

**Abstract Root Cause** (language-agnostic):
- **Namespace pollution**: Created identifier that collides with existing one
- **Incomplete refactoring**: Changed one part of system without updating dependents
- **No verification loop**: Made change → didn't verify → broke system
- **Context blindness**: Focused on creating file, ignored broader system state

**Should have been caught by** (missing checks):
1. `NAMESPACE_CHECK`: Before creating `config.py`, search for existing `config` identifiers
2. `DEPENDENCY_TRACE`: After creating file, trace what imports it
3. `POST_CHANGE_VALIDATION`: After any rename/move, verify imports still resolve
4. `SYSTEMATIC_SEARCH`: "Does this name conflict with anything?"

**Time to identify**: User reported  
**Time to fix**: ~1 minute (update one import)  
**Time that could have been saved**: 100% with namespace check

**Core Issue**: **Didn't verify system coherence after making changes**

---

**Key takeaways**: 
1. IMPORT_TRACE and ENVIRONMENT_CHECK are MANDATORY for nested structures
2. NAMESPACE_CHECK is MANDATORY before creating any file/function/class
3. DEPENDENCY_TRACE is MANDATORY after creating files that will be imported
4. POST_CHANGE_VALIDATION is MANDATORY after any refactoring

### BUG #3: NoneType Callable Error ❌ RUNTIME

**Error**: `TypeError: 'NoneType' object is not callable` in `ui/components/sliders.py`

**Location**: `Selector()` function passing `format_func=None` to `st.selectbox()`

**Root Cause - PARAMETER VALIDATION FAILURE**:
```
CREATED: Selector(format_func=None as default)
CALLED: st.selectbox(..., format_func=None)
STREAMLIT: Tries to call format_func(option) → crashes because None is not callable
RESULT: TypeError at runtime
```

**Why it happened - PLANNING GAPS**:
1. **No API_CONTRACT_CHECK**: Didn't verify what Streamlit expects for optional parameters
2. **No RUNTIME_ASSUMPTION_VALIDATION**: Assumed passing None is safe, didn't verify
3. **No TEST_EXECUTION**: Created component but didn't test it with default parameters
4. **Didn't read documentation**: Streamlit expects format_func to be omitted, not None

**Abstract Root Cause** (language-agnostic):
- **Invalid parameter passing**: Passed explicit None where library expects omitted parameter
- **API contract violation**: Didn't verify library's expectations for optional parameters
- **Untested code paths**: Created function but didn't test default parameter path
- **Assumption over verification**: Assumed None == omitted, but library treats them differently

**Should have been caught by** (missing checks):
1. `API_CONTRACT_CHECK`: Read Streamlit docs for st.selectbox format_func parameter
2. `PARAMETER_VALIDATION`: Check if None is acceptable or if parameter should be omitted
3. `TEST_DEFAULT_PATH`: Test function with all default parameters before integration
4. `DOCUMENTATION_REVIEW`: "What does the library expect for optional parameters?"

**Fix Applied**:
```python
# Before: Always passes format_func (even if None)
return st.selectbox(..., format_func=format_func)

# After: Only pass format_func if not None
kwargs = {... all required params ...}
if format_func is not None:
    kwargs["format_func"] = format_func
return st.selectbox(**kwargs)
```

**Time to identify**: User reported  
**Time to fix**: ~30 seconds (conditional parameter passing)  
**Time that could have been saved**: 100% with API contract check or simple test

**Core Issue**: **Didn't verify library's expectations for optional parameters**

3. DEPENDENCY_TRACE is MANDATORY after creating files that will be imported
