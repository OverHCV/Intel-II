## ARCHITECTURE DECISIONS & LEARNINGS

### What We Changed from Plan

**1. No Versioning Layer**
- **Planned**: experiments/ folder with JSON + pickle
- **Actual**: Session state + inline history table
- **Why**: Simpler, faster, sufficient for single-session
- **Tradeoff**: No cross-session persistence (acceptable for exam)

**2. No Template Inheritance**
- **Planned**: Abstract PageTemplate class
- **Actual**: Pattern copying between pages
- **Why**: Less abstraction overhead, more direct
- **Tradeoff**: Some code duplication (acceptable, <300 LOC per file)

**3. Inline Components**
- **Planned**: ui/patterns/ folder with reusable components
- **Actual**: Streamlit widgets directly in pages
- **Why**: Streamlit already provides good widgets
- **Tradeoff**: Less reusability (acceptable, fast to copy)

**4. Inline History**
- **Planned**: Separate history.py page
- **Actual**: History table at bottom of DT page
- **Why**: Better integration, see results + history together
- **Tradeoff**: Couples history to page (acceptable, only DT has history)

### What Worked Well

✅ **Data Layer**: Clean separation, easy to test, no bugs
✅ **State Management**: Centralized, predictable, works across pages
✅ **Feature Constants**: HUGE win, made everything readable
✅ **G1/G2 Toggles**: Excellent for demonstrating data leakage
✅ **Inline History**: Great UX, immediately compare experiments
✅ **Bug Tracking**: Caught and fixed data leakage early

### What Could Be Improved

⚠️ **Planning**: Over-architected initially (templates, versioning)
⚠️ **Scope**: Should have focused on one algorithm from start
⚠️ **Testing**: No unit tests (time constraint, acceptable for demo)

### Learnings for A2

1. ✅ Start with core algorithm implementation FIRST
2. ✅ Copy working patterns (DT page) instead of abstracting
3. ✅ Keep it simple (session state > persistence)
4. ✅ Feature constants are essential
5. ✅ Inline history > separate page

## NOTES & REFLECTIONS

### What Worked Well
- Clean data layer made everything downstream easier
- State management prevented many bugs
- Feature constants made UI readable
- Inline history better UX than separate page
- G1/G2 toggles great for experimentation

- Should have focused on one algorithm first
- Some pages exceeded 300 LOC (500+ is manageable)

### Learnings
- Simple > complex (session state > persistence)
- Inline > separate (history in page > separate page)
- Copy > abstract (pattern copying > template inheritance)
- Constants > magic strings (feature descriptions essential)
- Focus > breadth (complete DT > partial everything)
