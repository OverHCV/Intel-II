## ℹ️ HOT-RELOAD FEATURE

**Streamlit has built-in hot-reload!**

### To Enable:
1. Click the hamburger menu (☰) in top-right of Streamlit app
2. Go to "Settings"
3. Under "Development", enable "**Always rerun**"
4. Or: Configure in `.streamlit/config.toml`:
   ```toml
   [server]
   runOnSave = true
   ```

**How it works**: Streamlit watches your files and automatically reruns the app when you save changes.

## 🔍 Lessons Learned (For ParLang Framework)

### New Check Needed: FRAMEWORK_WIDGET_CHECK

**Purpose**: Verify framework-specific widget requirements

```
FRAMEWORK_WIDGET_CHECK IN(@ui_components, @framework_docs) OUT(widget_validation): {
  framework: "Streamlit",
  
  checks: {
    stateful_widgets: {
      rule: "All widgets with state need unique 'key' parameter",
      widgets: ["button", "checkbox", "slider", "selectbox", ...],
      validates: "No duplicate IDs"
    },
    conditional_rendering: {
      rule: "Widgets inside conditionals may render multiple times",
      validates: "Keys are unique across all render paths"
    }
  },
  
  IF (multiple_pages_with_similar_widgets) THEN {
    mandate: "Add descriptive keys: 'page_name_widget_purpose'",
    example: "svm_save_best_model vs ann_save_best_model"
  },
  
  why: "Framework generates IDs from (type, label, params) - identical widgets collide"
}
```

**When to use**: ALWAYS when creating UI components in framework-based apps

**Time cost**: 10 seconds per widget  
**Time saved**: Hours of debugging obscure framework errors
