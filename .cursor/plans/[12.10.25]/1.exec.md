**1. Non-Linear Classification Problem**
- Linear classifiers only work for linearly separable data
- Solution: Map to higher dimensional space φ: ℝⁿ → ℝᵐ where m > n
- Example: 2D circular data becomes linearly separable in 3D

**2. The Kernel Trick**
- Instead of computing φ(x) explicitly (expensive)
- Compute kernel k(a,b) = φ(a)ᵀφ(b) directly
- Saves computational cost while achieving same result
- Mathematical proof: polynomial kernel k(a,b) = (aᵀb)²

**3. Common Kernels**
- **Linear**: k(x,y) = xᵀy (no mapping, for linearly separable data)
- **Polynomial**: k(x,y) = (γ·xᵀy + r)^d (maps to polynomial space)
- **RBF (Gaussian)**: k(x,y) = exp(-γ||x-y||²) (infinite dimensional mapping)
- **Sigmoid**: k(x,y) = tanh(γ·xᵀy + r) (neural network-like)

**4. Parameters**
- **C (Regularization)**: Controls trade-off between margin and classification errors
  - Small C: Wide margin, more errors allowed (underfitting risk)
  - Large C: Narrow margin, fewer errors (overfitting risk)
  
- **gamma (γ)**: Kernel coefficient
  - Small γ: Far points have influence (smoother boundary)
  - Large γ: Only nearby points matter (complex boundary, overfitting risk)
  - 'scale': 1/(n_features * X.var())
  - 'auto': 1/n_features
  
- **degree (d)**: Polynomial degree (poly kernel only)
  - Higher degree: More complex decision boundary
  - Typically 2-5, higher risks overfitting

**5. Why Use Kernels?**
- Avoid explicit computation of high-dimensional mappings
- Enable non-linear classification efficiently
- RBF is most popular: works well for many problems without knowing data structure
