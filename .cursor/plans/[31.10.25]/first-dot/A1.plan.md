# Student Performance Analysis - Architecture Plan

> **Objective**: Complete third partial exam with modular, reusable architecture
> **Philosophy**: Clean separation (Logic | Data | UI), tech-agnostic design, pattern-first approach

---

## CONTEXT

**Problem Domain**: Educational data analysis (classification + clustering)

**Dataset**: Student Performance (UCI)
- 649 students, 33 attributes (mixed types)
- Target: G3 (0-20) → categorical transformation required
- Risk: G1, G2 correlation (data leakage)

**Deliverables**:
1. Decision Trees + rule analysis (0.9 pts)
2. Hierarchical clustering + J4 analysis (0.9 pts)  
3. K-means + comparison (0.7 pts)
4. Jupyter notebook + oral presentation (2.5 pts)

**Core Principle**: Build architecture where **next tab reuses layout pattern but swaps content/logic**

## ARCHITECTURE PRINCIPLES

```apl
VERB define_architecture_principles OUT(principles: obj) {
    
    separation_of_concerns: {
        rule: "Logic NEVER knows about UI, UI NEVER contains logic",
        layers: {
            LOGIC: "Pure algorithms, no display code",
            DATA: "Loading, transformation, validation",
            UI: "Display, interaction, layout patterns"
        },
        communication: "Through well-defined contracts (interfaces)"
    }
    
    modularity: {
        rule: "Each module < 300 LOC, single responsibility",
        file_organization: {
            by_layer: "logic/, data/, ui/ - NOT by feature",
            by_responsibility: "One file = one clear purpose"
        }
    }
    
    reusability: {
        WHAT_CHANGES: {
            logic: "Algorithm implementations vary per task",
            content: "Visualizations, metrics, parameters differ",
            data_transforms: "Preprocessing may differ"
        },
        WHAT_STAYS_SAME: {
            layout_patterns: "Two-column, tabs, controls structure",
            ui_components: "Buttons, sliders, displays",
            data_pipeline: "Load → validate → transform flow",
            state_management: "How state is stored/accessed"
        },
        pattern: "Define TEMPLATE with SLOTS for variable content"
    }
    
    testability: {
        rule: "Each layer independently testable",
        logic_tests: "Test algorithms with mock data",
        data_tests: "Test transformations with sample inputs",
        ui_tests: "Test rendering with mock logic"
    }
    
    extensibility: {
        rule: "Adding new task = plug new logic into existing structure",
        example: "Task 4 would reuse ALL infrastructure, only swap logic module"
    }
}
```

## LAYER DEFINITIONS

### Layer 1: DATA (Pure Transformation)

```apl
VERB define_data_layer OUT(@data/) {
    
    responsibility: "Load, validate, transform data - NO business logic",
    
    INTERFACE DataProvider {
        VERB load IN(source: str, options: obj) OUT(raw_data: dataframe) 
        VERB validate IN(raw_data: dataframe) OUT(is_valid: bool, errors: list)
        VERB transform IN(raw_data: dataframe, strategy: obj) OUT(processed_data: obj)
    }
    
    modules: {
        loader_py: {
            purpose: "File I/O only",
            functions: [
                "load_csv(filepath) → dataframe",
                "load_multiple(files) → merged_dataframe",
                "save_cache(data, key) → None"
            ],
            no_logic: "Just read/write, no decisions"
        },
        
        validator_py: {
            purpose: "Data quality checks",
            functions: [
                "check_schema(df, expected_cols) → bool",
                "check_missing(df, tolerance) → bool",
                "check_ranges(df, constraints) → bool"
            ]
        },
        
        transformer_py: {
            purpose: "Data transformations",
            functions: [
                "encode_categorical(df, method) → encoded_df",
                "scale_numerical(df, method) → scaled_df",
                "engineer_target(df, strategy) → target_array",
                "split_features_target(df, target_col) → X, y"
            ],
            parametrized: "Strategy pattern - caller decides HOW"
        }
    },
    
    contract: {
        input: "File path + transformation options",
        output: "X (features), y (target), metadata",
        guarantees: [
            "X.shape[0] == y.shape[0]",
            "No NaN in output",
            "Metadata contains feature names, types, stats"
        ]
    }
}
```

### Layer 2: LOGIC (Pure Algorithms)

```apl
VERB define_logic_layer OUT(@logic/) {
    
    responsibility: "Implement ML algorithms - NO data loading, NO display",
    
    INTERFACE Algorithm {
        VERB fit IN(X: array, y: array, params: obj) OUT(model: obj)
        VERB predict IN(model: obj, X: array) OUT(predictions: array)
        VERB evaluate IN(y_true: array, y_pred: array) OUT(metrics: obj)
        VERB explain IN(model: obj) OUT(explanation: obj)
    }
    
    modules: {
        decision_tree_py: {
            purpose: "Tree algorithms",
            functions: [
                "train_cart(X, y, hyperparams) → model",
                "extract_rules(model) → list[rule_obj]",
                "rank_rules(rules, criteria) → sorted_rules",
                "cross_validate(X, y, model_fn, k) → cv_scores"
            ],
            pure: "Takes arrays, returns objects - no side effects"
        },
        
        clustering_py: {
            purpose: "Clustering algorithms",
            functions: [
                "hierarchical(X, linkage) → linkage_matrix",
                "kmeans(X, k, params) → model, labels",
                "calculate_j4(X, labels) → float",
                "compare_clusterings(labels1, labels2) → agreement_metrics"
            ]
        },
        
        analysis_py: {
            purpose: "Result analysis",
            functions: [
                "find_optimal_k(linkage_matrix, X, criterion) → k, scores",
                "analyze_rules(rules) → insights",
                "cluster_statistics(X, labels) → stats_per_cluster"
            ]
        }
    },
    
    contract: {
        input: "Numpy arrays + parameters",
        output: "Models, predictions, metrics (no formatting)",
        guarantees: [
            "Deterministic (same input → same output)",
            "No file I/O",
            "No print statements",
            "Returns data, not strings"
        ]
    }
}
```

### Layer 3: UI (Pure Display)

```apl
VERB define_ui_layer OUT(@ui/) {
    
    responsibility: "Display results, handle interaction - NO algorithms",
    
    INTERFACE UIComponent {
        VERB render IN(data: obj, layout: obj) OUT(displayed: bool)
        VERB handle_interaction IN(event: obj) OUT(action: obj)
    }
    
    architecture: {
        pattern_library: {
            TwoColumnLayout: {
                slots: ["controls", "display"],
                reusable: true,
                variant: "Ratio adjustable (1:2, 1:3, etc.)"
            },
            
            TabContainer: {
                slots: ["tab_contents[]"],
                reusable: true,
                variant: "Number of tabs variable"
            },
            
            ControlPanel: {
                slots: ["widgets[]"],
                reusable: true,
                variant: "Widget types configurable"
            },
            
            ResultDisplay: {
                slots: ["visualization", "metrics", "insights"],
                reusable: true,
                variant: "Content types variable"
            }
        },
        
        page_template: {
            structure: `
                TwoColumnLayout(
                    left: ControlPanel(
                        widgets: [param_selectors, train_button, status]
                    ),
                    right: TabContainer(
                        tabs: [
                            ResultDisplay(viz1, metrics1),
                            ResultDisplay(viz2, metrics2)
                        ]
                    )
                )
            `,
            swappable: [
                "param_selectors - different per algorithm",
                "viz1, viz2 - different visualizations",
                "metrics1, metrics2 - different metrics"
            ],
            constant: "Layout structure, interaction flow"
        }
    },
    
    modules: {
        components: {
            layouts_py: "TwoColumn, Tabs, Grid",
            controls_py: "Sliders, Selects, Buttons",
            displays_py: "MetricCard, Table, PlotContainer"
        },
        
        pages: {
            template_py: {
                purpose: "Abstract page template",
                methods: [
                    "render_controls() - override per page",
                    "render_results() - override per page",
                    "handle_train_click() - calls logic layer"
                ]
            },
            
            decision_tree_page_py: "Extends template, provides content",
            hierarchical_page_py: "Extends template, provides content",
            kmeans_page_py: "Extends template, provides content"
        },
        
        orchestrator_py: {
            purpose: "Wire layers together",
            pattern: `
                user_clicks_train()
                  → gather_params_from_ui()
                  → data = DataProvider.load_and_transform(params)
                  → results = Algorithm.fit_and_evaluate(data, params)
                  → UIComponent.render(results)
            `
        }
    },
    
    contract: {
        input: "Data objects from logic layer (models, metrics, arrays)",
        output: "Visual representation",
        responsibilities: [
            "Format data for display",
            "Handle user interactions",
            "Manage UI state",
            "Call logic layer functions"
        ],
        prohibited: [
            "Implement algorithms",
            "Load raw files",
            "Transform data"
        ]
    }
}
```

## IMPLEMENTATION STRATEGY

```apl
VERB plan_implementation OUT(strategy: obj) {
    
    phase1_contracts: {
        purpose: "Define interfaces BEFORE implementation",
        deliverables: [
            "DataProvider interface spec",

            task1: "decision_tree.py + analysis.py",

        purpose: "Build pattern library",
```
