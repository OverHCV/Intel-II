# CRITICAL FIX #4: Import Order & Path Changes

## Error
`ModuleNotFoundError: No module named 'settings'`

## Root Cause Analysis

**What happened**:
1. User moved `config.py` from project root → `settings/config.py`
2. Changed one import in `ui/app.py` line 10: `from settings.config import ...`
3. But forgot to:
   - Update 8 other files that import from config
   - Move `sys.path` manipulation BEFORE imports (it was AFTER!)

**Why it failed**:
```python
# ❌ WRONG ORDER (before fix):
import streamlit as st
from settings.config import CONF, Keys  # ← Tries to import
from ui.pages import ...                # ← Tries to import

# Add project root to Python path
sys.path.insert(0, str(project_root))  # ← TOO LATE!
```

**Import resolution**:
- Line 10-12: Python tries to import `settings.config`
- But `settings/` is not in `sys.path` yet!
- Line 14-16: sys.path gets modified (but too late, imports already failed)

## Fix Applied

### 1. Fixed import order in `ui/app.py`
```python
# ✅ CORRECT ORDER:
import sys
from pathlib import Path

# Add project root FIRST (before any local imports)
project_root = Path(__file__).parent.parent
sys.path.insert(0, str(project_root))

# Now imports work
import streamlit as st
from settings.config import CONF, Keys
from ui.pages import ...
```

### 2. Updated all 8 files that import from config

**Files updated**:
1. `ui/app.py` - Fixed import order
2. `ui/pages/conf.py` - `from config import` → `from settings.config import`
3. `ui/utils/state_manager.py` - Updated import
4. `ui/pages/ann.py` - Updated import
5. `ui/pages/svm.py` - Updated import
6. `funcs/visualizers.py` - Updated import
7. `ui/utils/data_loader.py` - Updated import
8. `main.py` - Updated import
9. `funcs/evaluator.py` - Updated import (was `from config import Keys`)

## Why This Happened

**Missing Check**: `POST_CHANGE_VALIDATION`

**Should have done**:
```
REFACTOR: Move config.py → settings/config.py

POST_CHANGE_VALIDATION {
  1. DEPENDENCY_TRACE: Find all files importing from config
     → grep -r "from config import" .
     → Found 8 files
  
  2. UPDATE_ALL_DEPENDENTS: Change imports in all 8 files
  
  3. TEST_EXECUTION: Run app to verify
     → Would catch import errors immediately
}
```

**Actual process**:
- User moved file
- User updated ONE import manually
- Didn't trace dependents
- Didn't validate after change
- System broke

## Abstract Pattern (Language-Agnostic)

**Incomplete Refactoring Strike 2**:
1. Move/rename module
2. Update SOME references
3. Forget to update ALL references
4. Don't validate after change
5. Runtime failure

**Plus: Import Order Violation**:
- Tried to use path manipulation after imports run
- Order matters in sequential execution!

## Prevention

**MANDATORY checks for ANY file move/rename**:
1. `DEPENDENCY_TRACE`: Find all references (`grep -r "from oldpath import"`)
2. `UPDATE_ALL_REFS`: Change every reference found
3. `IMPORT_ORDER_CHECK`: Verify sys.path mods happen BEFORE local imports
4. `POST_CHANGE_VALIDATION`: Run app, catch errors immediately

**Golden Rule #6**: NEVER move/rename files without tracing ALL dependents

## Time Cost
- **Check + update all refs**: ~2 minutes
- **Debugging without checks**: 15+ minutes
- **Ratio**: 7.5x time saved with checks
